# 快速推送指南

## 问题：批处理文件无法输入密码

如果您在使用 `添加密钥并推送.bat` 时遇到密码无法输入的问题，请使用以下方法：

## 解决方案

### 方法一：使用 PowerShell 脚本（最简单）

**双击运行：`运行PowerShell脚本.bat`**

这个脚本会调用 PowerShell 版本，可以正常输入密码。

### 方法二：直接在 PowerShell 中执行

1. **打开 PowerShell**（按 Win+X，选择 "Windows PowerShell"）

2. **切换到项目目录**：
   ```powershell
   cd "C:\Users\PC\Desktop\简历上传"
   ```

3. **添加 SSH 密钥**（会提示输入密码）：
   ```powershell
   ssh-add "$env:USERPROFILE\.ssh\id_ed25519"
   ```
   
   **注意**：输入密码时不会显示字符，这是正常的。直接输入并按回车即可。

4. **验证密钥已添加**：
   ```powershell
   ssh-add -l
   ```
   
   应该看到类似输出：
   ```
   256 SHA256:hMMjFPOhcINufN98RmIpO7swZiNrlzWB4V4/KLSlv6c 1057177609@qq.com (ED25519)
   ```

5. **推送代码**：
   ```powershell
   git push origin main
   ```

### 方法三：使用命令提示符（CMD）

1. **打开命令提示符**（按 Win+R，输入 `cmd`，回车）

2. **切换到项目目录**：
   ```cmd
   cd "C:\Users\PC\Desktop\简历上传"
   ```

3. **添加 SSH 密钥**：
   ```cmd
   ssh-add "%USERPROFILE%\.ssh\id_ed25519"
   ```
   
   **注意**：输入密码时不会显示字符，这是正常的。

4. **推送代码**：
   ```cmd
   git push origin main
   ```

## 为什么批处理文件无法输入密码？

Windows 批处理文件在执行某些交互式命令（如 `ssh-add`）时，密码输入可能不会正常显示。这是 Windows 终端的一个已知限制。

使用 PowerShell 或命令提示符可以避免这个问题。

## 推送成功后

一旦密钥成功添加到 ssh-agent，以后的推送就不需要再输入密码了！

只要 ssh-agent 服务在运行（我们已经设置为自动启动），密钥就会一直保持在内存中。
