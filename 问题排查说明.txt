   输入 python app.py 后没有任何输出的问题排查
═══════════════════════════════════════════════════════════

【问题现象】
在命令提示符中输入 "python app.py" 后，没有任何输出，直接返回命令提示符。

【可能的原因】

1. 程序在导入阶段出错，但错误被忽略了
2. OCR初始化失败导致程序退出
3. 缺少必要的依赖包
4. Python环境配置问题

【解决方案】

方法1：使用快速启动脚本（推荐）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  双击运行：快速启动.bat
  这个脚本会自动捕获错误并显示详细信息

方法2：使用简单启动脚本
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  在命令提示符中运行：
    python 简单启动.py
  这会禁用OCR并显示所有错误信息

方法3：使用详细诊断脚本
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  在命令提示符中运行：
    python 详细诊断.py
  这会逐步检查每个模块，找出具体问题

方法4：手动测试导入
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  在命令提示符中逐步运行以下命令：

  1. 测试config模块：
     python -c "from config import Config; print('config OK')"

  2. 测试models模块：
     python -c "from models import get_db_session; print('models OK')"

  3. 测试app模块：
     python -c "from app import app; print('app OK')"

  如果某个步骤失败，会显示具体错误信息。

方法5：检查Python环境
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  运行以下命令检查：

  python --version        # 检查Python版本
  pip list                # 查看已安装的包
  pip install -r requirements.txt  # 安装所有依赖

方法6：临时禁用OCR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  OCR初始化可能导致程序崩溃。可以临时禁用：

  1. 打开 config.py 文件
  2. 找到 OCR_ENABLED 配置
  3. 设置为 False：
     OCR_ENABLED = False

  4. 然后重新运行程序

【常见错误及解决】

错误1：ModuleNotFoundError: No module named 'flask'
解决：运行 pip install flask

错误2：ModuleNotFoundError: No module named 'sqlalchemy'
解决：运行 pip install sqlalchemy

错误3：OCR初始化失败
解决：可以禁用OCR（见方法6），或安装OCR库：
      pip install easyocr

错误4：端口5000被占用
解决：修改 app.py 中的端口号，或结束占用进程

【获取详细错误信息】

如果以上方法都无效，请：

1. 运行诊断脚本并保存输出：
   python 详细诊断.py > 错误日志.txt

2. 查看生成的 错误日志.txt 文件

3. 将错误信息提供给开发者
