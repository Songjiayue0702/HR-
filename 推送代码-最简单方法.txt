您刚才遇到的问题：

1. ❌ 错误：使用了 `ssh-ed25519` 作为命令
   → ssh-ed25519 是密钥类型，不是命令

2. ❌ 错误：`$env: USERPROFILE` 中间有空格
   → 正确写法是 `$env:USERPROFILE`（没有空格）

3. ❌ 密码输入错误
   → 如果多次提示 "Bad passphrase"，可能是密码记错了

方法二：手动执行（在 PowerShell 中）

1. 打开 PowerShell

2. 复制并执行以下命令（注意：一行一行执行）：

   cd "C:\Users\PC\Desktop\简历上传"
   
   ssh-add "$env:USERPROFILE\.ssh\id_ed25519"
   
   git push origin main

重要提示：
- $env: 和 USERPROFILE 之间没有空格！
- 输入密码时不显示字符是正常的
- 如果密码错误，请检查密码是否正确

═══════════════════════════════════════════════════════
如果忘记密码怎么办？
═══════════════════════════════════════════════════════

如果确实忘记了 SSH 密钥密码，有两个选择：

选择 1: 重新生成新的 SSH 密钥（推荐）
  1. 生成新的密钥（不设置密码）
  2. 将新公钥添加到 GitHub
  3. 更新本地使用新密钥

选择 2: 使用 HTTPS 方式推送（需要 Personal Access Token）
  1. 在 GitHub 生成 Personal Access Token
  2. 使用 HTTPS URL 推送
  3. 使用 token 作为密码
