# 🚀 立即启动指南

## 问题：输入 `python app.py` 后没有任何输出

这说明程序可能在启动时遇到了错误，但错误信息没有显示出来。

## ✅ 解决方案

### 方法1：使用批处理文件启动（推荐）

**双击运行以下文件之一：**

1. **`最简启动.bat`** - 最简单的启动方式
2. **`一键启动.bat`** - 带详细诊断的启动
3. **`启动并查看输出.bat`** - 显示所有输出信息

这些批处理文件会：
- 自动检查环境
- 显示所有错误信息
- 如果启动失败，会显示具体原因

### 方法2：在命令提示符中手动启动

1. **打开命令提示符**（按 Win + R，输入 `cmd`，按回车）

2. **切换到项目目录**
   ```cmd
   cd "C:\Users\PC\Desktop\简历上传"
   ```

3. **禁用OCR（避免初始化问题）**
   ```cmd
   set OCR_ENABLED=false
   ```

4. **运行程序**
   ```cmd
   python app.py
   ```

5. **查看输出**
   - 如果看到 `* Running on http://127.0.0.1:5000`，说明启动成功
   - 如果有错误信息，请查看并解决

### 方法3：使用Python脚本启动

运行诊断脚本：
```cmd
python 直接启动并显示错误.py
```

或者运行详细诊断：
```cmd
python 详细诊断.py
```

## 🔍 常见问题排查

### 问题1：程序立即退出，没有任何输出

**可能原因：**
- 导入模块时出错
- OCR初始化失败
- 缺少依赖包

**解决方法：**
1. 运行诊断脚本查看错误：
   ```cmd
   python 详细诊断.py
   ```

2. 或者尝试禁用OCR：
   ```cmd
   set OCR_ENABLED=false
   python app.py
   ```

### 问题2：端口5000被占用

**解决方法：**
```cmd
# 查看占用端口的进程
netstat -ano | findstr :5000

# 结束进程（替换PID为实际进程ID）
taskkill /PID <进程ID> /F
```

或者修改端口：
- 打开 `app.py`
- 找到第2774行：`port = 5000`
- 改为其他端口，如：`port = 5001`

### 问题3：缺少依赖包

**解决方法：**
```cmd
pip install -r requirements.txt
```

如果安装失败，可以逐个安装：
```cmd
pip install flask
pip install sqlalchemy
pip install PyPDF2
pip install python-docx
pip install openpyxl
pip install reportlab
```

## 📝 重要说明

1. **我已经修改了config.py，默认禁用OCR**，这样可以避免OCR初始化导致的问题。

2. **如果程序启动成功**，你会看到类似这样的输出：
   ```
   ==================================================
   智能简历数据库系统
   ==================================================
   服务器地址: http://127.0.0.1:5000
   局域网地址: http://0.0.0.0:5000
   ==================================================
   按 Ctrl+C 停止服务器
   ==================================================

   * Running on all addresses (0.0.0.0)
   * Running on http://127.0.0.1:5000
   ```

3. **如果启动成功但没有看到输出**，可能是输出被重定向了。尝试：
   ```cmd
   python -u app.py
   ```
   `-u` 参数会强制立即输出，不缓冲。
